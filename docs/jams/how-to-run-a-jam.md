# How to Run a Jam

A lightweight process for recurring Indie Cake community jams.

## Jam lifecycle

1. Announce theme and constraints
2. Open submissions
3. Run playtest window
4. Curate featured entries
5. Publish results and highlights

## Recommended timeline

- Day 1: Theme reveal
- Day 1-6: Build window
- Day 7: Submission close and showcase

## Suggested constraints

- Single objective
- 8 seconds or less
- Works on desktop and mobile
- No external account requirement
- Prefer a pinned engine version for hosted entries

## Submission format

Each entry includes:

- game folder in `games/`
- `indiecake.json`
- short README
- optional thumbnail

## Judging criteria

- Clarity
- Fun in short bursts
- Theme alignment
- Technical stability
- Creativity

## Organizer checklist

- Publish jam-template.md for each new jam
- Create jam tag convention
- Verify metadata and links
- Update `game-index.json` with jam entries
- Point creators to [Engine Quick Reference](../engine-quick-reference.md)
- Ask creators to include engineVersion in metadata when engine is used
